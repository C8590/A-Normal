# IMPORT_SPEC

The import layer creates versioned local data directories for research runs. It prevents new data batches from overwriting `data/sample/ashare_alpha` and gives each batch a `source_name` and `data_version`.

This task does not connect to real external data sources. It only copies local CSV files, validates them, and records metadata.

## Directory Layout

Imported batches are stored as:

```text
data/imports/{source_name}/{data_version}/
  stock_master.csv
  daily_bar.csv
  financial_summary.csv
  announcement_event.csv
  import_manifest.json
  validation_report.json
  data_snapshot.json
```

`data_snapshot.json` is written only when validation passes.

## Import Manifest

`import_manifest.json` records:

- `import_id`
- `source_name`
- `data_version`
- `created_at`
- `source_data_dir`
- `target_data_dir`
- `config_dir`
- copied file metadata with row counts and SHA-256 hashes
- validation status and issue counts
- snapshot id and path when available
- final status: `SUCCESS` or `FAILED`
- optional notes

`source_name` and `data_version` must be non-empty. A failed import includes a Chinese-readable `error_message`.

## Validation Report

`validation_report.json` is the serialized `DataValidationReport` from `LocalCsvAdapter.validate_all()`:

- `passed`
- `errors`
- `warnings`
- `row_counts`

If validation fails after files are copied, the import is marked `FAILED`, but both `import_manifest.json` and `validation_report.json` are still saved.

## Data Snapshot

`data_snapshot.json` is generated by the point-in-time audit module. It records:

- snapshot id
- created time
- source and version
- data/config directories
- row counts
- minimum and maximum dates for each dataset

The snapshot is metadata only; it does not modify source data.

## Version Naming

`source_name` is normalized:

- lower case
- spaces become underscores
- unsupported characters become underscores
- only letters, numbers, underscores, and hyphens remain

`data_version` may contain letters, numbers, underscores, hyphens, and dots. It cannot be empty, cannot contain path separators, and cannot contain `..`.

If no version is supplied, the importer generates a timestamp such as:

```text
v20260320_153000
```

## Commands

Import a local CSV batch:

```bash
python -m ashare_alpha import-data \
  --source-name local_csv \
  --source-data-dir data/sample/ashare_alpha
```

Use an explicit version:

```bash
python -m ashare_alpha import-data \
  --source-name local_csv \
  --source-data-dir data/sample/ashare_alpha \
  --data-version sample_v1
```

Overwrite an existing version:

```bash
python -m ashare_alpha import-data \
  --source-name local_csv \
  --source-data-dir data/sample/ashare_alpha \
  --data-version sample_v1 \
  --overwrite \
  --format json
```

Run a quality report after a successful import:

```bash
python -m ashare_alpha import-data \
  --source-name local_csv \
  --source-data-dir data/sample/ashare_alpha \
  --data-version sample_v1 \
  --overwrite \
  --quality-report
```

`--quality-report` is disabled by default. When enabled, it writes `quality_report.json`, `quality_report.md`, and `quality_issues.csv` into the imported version directory. If quality errors are found, the import manifest can remain `SUCCESS` when validation passed, but `notes` will include a reminder to inspect the quality report.

List imports:

```bash
python -m ashare_alpha list-imports
python -m ashare_alpha list-imports --format json
```

Inspect one import:

```bash
python -m ashare_alpha inspect-import --source-name local_csv --data-version sample_v1
```

## Use Imported Data

Imported directories are ordinary `LocalCsvAdapter` data directories:

```bash
python -m ashare_alpha validate-data \
  --data-dir data/imports/local_csv/sample_v1
```

They can also be used in the daily research pipeline:

```bash
python -m ashare_alpha run-pipeline \
  --date 2026-03-20 \
  --data-dir data/imports/local_csv/sample_v1 \
  --audit-leakage
```

## Boundaries

The import layer does not call external APIs, scrape websites, connect to brokers, or place orders. It does not change `LocalCsvAdapter`, scoring, backtesting, probability models, or pipeline defaults. The system remains an offline research system and is not investment advice.
