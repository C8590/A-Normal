# Data Spec

## Principle

The MVP uses local CSV files only. It does not scrape websites, call broker APIs, submit orders, or require secrets.

## Sample Data Location

Default sample data lives in:

```text
data/sample/ashare_alpha/
```

Required files:

- `stock_master.csv`
- `daily_bar.csv`
- `financial_summary.csv`
- `announcement_event.csv`

## Tables

### stock_master

Fields:

- `ts_code`
- `symbol`
- `name`
- `exchange`: one of `sse`, `szse`, `bse`
- `board`: one of `main`, `chinext`, `star`, `bse`
- `industry`
- `list_date`
- `delist_date`
- `is_st`
- `is_star_st`
- `is_suspended`
- `is_delisting_risk`

### daily_bar

Fields:

- `trade_date`
- `ts_code`
- `open`
- `high`
- `low`
- `close`
- `pre_close`
- `volume`
- `amount`
- `turnover_rate`
- `limit_up`
- `limit_down`
- `is_trading`

Validation requires non-negative prices, volume, and amount. `high` must be greater than or equal to `low`, `open`, and `close`. Trading rows must have positive `amount`.

### financial_summary

Fields:

- `report_date`
- `publish_date`
- `ts_code`
- `revenue_yoy`
- `profit_yoy`
- `net_profit_yoy`
- `roe`
- `gross_margin`
- `debt_to_asset`
- `operating_cashflow_to_profit`
- `goodwill_to_equity`

Financial fields may be missing. `publish_date` must not be earlier than `report_date`.

### announcement_event

Fields:

- `event_time`
- `ts_code`
- `title`
- `source`
- `event_type`
- `event_direction`: one of `positive`, `negative`, `neutral`
- `event_strength`: 0 to 1
- `event_risk_level`: one of `low`, `medium`, `high`
- `raw_text`

The first MVP event types are `earnings_positive`, `earnings_negative`, `buyback`, `shareholder_increase`, `shareholder_reduce`, `regulatory_penalty`, `investigation`, `litigation`, `major_contract`, `equity_pledge`, `unlock_shares`, and `unknown`.

## Validation Command

```bash
python -m ashare_alpha validate-data
python -m ashare_alpha validate-data --data-dir data/sample/ashare_alpha
python -m ashare_alpha validate-data --format json
```

The command prints row counts, warnings, and errors. It returns a non-zero exit code when validation fails.

## Adapter Boundary

`ashare_alpha.data.adapters.DataAdapter` defines the interface for loading normalized data. The current implementation is `LocalCsvAdapter`; future external adapters can be added later, but this MVP task does not call external APIs.

## Daily Universe Output

`universe_daily` is generated by:

```bash
python -m ashare_alpha build-universe --date 2026-03-20
python -m ashare_alpha build-universe --date 2026-03-20 --format json
```

The CSV output defaults to:

```text
outputs/universe/universe_daily_YYYY-MM-DD.csv
```

Each row includes:

- identity fields: `trade_date`, `ts_code`, `symbol`, `name`, `exchange`, `board`, `industry`
- decision fields: `is_allowed`, `exclude_reasons`, `exclude_reason_text`
- computed fields: `listing_days`, `latest_close`, `one_lot_value`, `avg_amount_20d`, `trading_days_20d`
- score fields: `liquidity_score`, `risk_score`
- event fields: `has_recent_negative_event`, `recent_negative_event_count`, `latest_negative_event_title`

The builder uses only data on or before the target `trade_date`.

## Exclusion Reasons

- `BOARD_NOT_ALLOWED`: 所属板块不在第一版允许范围内
- `BOARD_EXCLUDED`: 所属板块在第一版排除范围内
- `IS_ST`: ST 股票，第一版禁止进入股票池
- `IS_STAR_ST`: *ST 股票，第一版禁止进入股票池
- `DELISTING_RISK`: 存在退市风险，第一版禁止进入股票池
- `SUSPENDED`: 股票处于停牌状态，第一版禁止进入股票池
- `LISTING_DAYS_TOO_SHORT`: 上市天数低于配置要求
- `INSUFFICIENT_DAILY_BARS`: 近 20 日行情数据不足
- `LOW_AVG_AMOUNT_20D`: 近 20 日平均成交额低于配置阈值
- `TOO_EXPENSIVE_FOR_CAPITAL`: 1 手金额超过当前本金允许的单票上限
- `RECENT_NEGATIVE_EVENT`: 近期存在负面公告或高风险事件
- `MISSING_LATEST_BAR`: 目标交易日缺少行情数据
- `NOT_TRADING_ON_DATE`: 目标交易日不可交易
